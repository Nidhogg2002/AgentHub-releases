# Agent Hub 安装与使用指南

感谢您下载 Agent Hub！本指南将帮助您快速完成软件的安装与初始配置。

## 📥 安装说明

1. **获取安装包**
   请在当前 Release 页面下载对应的安装程序：
   - Windows 用户请下载：`Agent Hub-1.5.1-setup.exe` (或更高版本)

2. **执行安装**
   双击下载的 `.exe` 安装文件，按照提示完成安装。软件将自动在桌面创建快捷方式。

3. **智能热更新**
   从 `v1.5.1` 起，Agent Hub 已集成静默热更新功能。以后当我们在 GitHub 发布了新版本时：
   - 软件会在后台自动下载新版本。
   - 下载完成后，进入软件的 **设置 -> 关于软件**，点击【重启并更新】即可瞬间升级到最新版本，无需再次手动下载安装包！

## 🚀 首次使用与配置

Agent Hub 旨在统一管理您电脑上现有的各种大模型智能体（CLI Agent）。在开始使用前，请确保您已经了解以下核心概念：

### 1. 检测本地 Agent
Agent Hub 本身是一个**桌面控制终端**，它会自动扫描您电脑上是否安装了对应的核心智能体：
- **Claude Code** (由 Anthropic 官方提供)
- **Codex**
- **AntiGravity**

软件启动时，如果没有检测到这些 Agent，相关功能可能会处于不可用状态。请确保您已经在系统环境（如 Node.js 全局环境）中成功安装并初始化过这些命令行工具。

### 2. 网络代理配置 (必读)
如果您所在的网络环境需要代理才能访问外部 API（如 OpenAI、Anthropic 等大模型接口）：
- 请确保在启动 Agent Hub 之前，系统级环境变量中配置好了 `http_proxy` 或 `https_proxy`（很多代理软件开启“系统代理”模式会自动设置环境变量，或者您可以在终端环境中手动设定后再打开软件）。
- **优化原理解释**：Agent Hub 为了帮您节省 Token 花费、缓存上下文，并在后台拦截了 Agent 的发信请求（在 `127.0.0.1:15721` 运行了一个本地中转）。因此软件会自动穿透读取您的系统代理环境变量来代发网络请求。

### 3. API 密钥与模型中心
为了获得最佳体验，您可以进入软件右上角的 **设置 -> 模型提供商** 面板：
- 添加您自己的 OpenAI 或 Anthropic API Key。
- 配置自定义的中转/兼容接口（Base URL）。
- 这个配置会**直接覆盖**各个智能体的默认模型选择。

## 🛡️ 常见问题与排错 (FAQ)

**Q1：安装时被 Windows Defender 拦截报毒？**
> 因为开源软件暂无昂贵的微软数字签名证书，Windows 可能会弹出蓝色的安全警告 (SmartScreen)。请点击 **“更多信息”** -> **“仍要运行”** 即可正常安装。

**Q2：提示 “无法连接到 Codex” 或网络超时？**
> 这通常是因为网络未联通。请检查您的科学上网工具是否正常工作，且已在系统中正确注入了全局的 `HTTP_PROXY` 环境变量。

**Q3：Agent Hub 能完全离线工作吗？**
> Agent Hub 的 UI 界面和会话存储（SQLite）是完全本地离线的。但它唤起的 AI 引擎在思考时仍需要连接云端大模型接口。
